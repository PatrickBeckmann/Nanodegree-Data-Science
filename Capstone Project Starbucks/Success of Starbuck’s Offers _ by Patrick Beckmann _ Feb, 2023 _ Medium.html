Success of Starbuck’s Offers

Overview

For the project I trained 6 different models used for binary classification to predict whether an offer made to a customer was successful and led to a transaction being made. My best performing model was a Support Vector Machine with RBF and it achieved an accuracy of nearly 80%. For future optimization I will need to train a model that is able to give a more diverse prediction. Right now, I can only predict the success of an offer. If I want to personalize a campaign to best suit a specific customer the model needs further improvements.

Description

As part of my Udacity Data Science Program I analysed a dataset with information about different Starbuck’s offers for their customers provided in the mobile reward app. The data is simulated and mimicing the behavior of customers in the app.

The goal of the project was to forecast whether an offer will be successful and the customer does a transaction based on it or not.
Since this is a binary classification — the offer is either successful or not, I trained several models to predict the success of an offer made to a customer.
Using GridSearchCV where possible I tried to optimize my parameters and improve the score. I measured the success of the model with the accuracy and the F1-score. The emphasis would be on the F1-score since it is the harmonic mean of recall and precision and takes both Type I and Type II errors into account.

Dataset

I analysed datasets provided by Starbucks about simulated data mimicing customer behavior on the Starbucks rewards mobile app. The data contains three different files about the customers, the offers itself and the transactions made. The “profile” dataset contains demographic data for each customer. The “portfolio” dataset contains the ids of each offer as well as meta data such as duration, type, difficulty, etc. The “transcript” data contains the records of the transactions being made as well as information about which offers has been viewed and completed.

Exploratory analysis and preprocessing

There are ten different offers with three types in the dataset: BOGO (Buy Oe Get One), a Discount or an informational offer and the offers have different difficulties. The difficulty of an offer is the amount that has to be spent to activate the offer. The reward of the offer would be the value that the customer would save or get from an offer.

Looking at the customers we have 8484 male customers, 6129 female customers and 212 others.

They are distributed over an age betwen 18 and 101and an income between 30.000 and 120.000 thousand Dollars.

Since I wanted to analyse the success of an offer, I merged the dataset and hat to create a measure for the success.
Because there transactions without the offer being viewed I couldn’t just use a successful transaction as an indicator. I multiplied the successful transaction with the value for an offer being viewed and could split the data like this in successful offers and unsuccessful offers.

To use the information provided by the meta data of the offers, I generated dummy variables for the different channels on which the offers were provided as well as the type of offer (BOGO, discount, informational).

I dealt with outliers, such as many ages at 118 and filled missing data for age and income using the median. Both distributions had a light right skew and therefore the median was better suited than the mean.

Modelling & Predicting the Success of an Offer

For the prediction I used different models for binary Classification. I tried to predict if an offer would be successful or not.
To compare the models I used the accuracy and the F1-Score. I put more emphasis on the F1-Score because it is the harmonic mean of recall and precision and takes both Type I and Type II errors into account.

I started with a Random Forest Classifier. But with an Accuracy ov only about 6.5% this model was not suited.

Instead I tried a Logistic Regression model. Since I had an imbalance between successful and unsuccessful offers, I sapled more successful offers until they were even.

For the Logistic Regression I could only use independend variables, so I tested for that and had to delete the email column from the dataset.
I used the MinMax Scaler and got an accuracy of almost 76% and a F1-Score of 0.77. I tried to improve the model by using GridSearchCV to optimize the parameters. But that did not bring any improvement.

The thrid model I used was Support Vector Machines (SVM). I first trained with the linear kernel and C at 1. The model achieved an accuracy of 73.61% and a F1 Score of 0.77.
I changed the kernel to the Radial Basic Function (RBF) an achieved an accuracy of 77.89% and a F1-score of 0.79.

Using GridSearchCV for the optimization of parameters would have taken too long for the SVM. Therefore I only changed the prarameter C to 100 and trained the model again. This last SVM model achieved an accuracy of 78.63% with an F1-score of 0.7965. therefore it was the best of the three SVM models.

The fourth model I used was a Linear Discriminant Analysis (LDA). This model is sensible to outliers. That is why I used the tukey rule to remove the remaining outliers that I didn’t already remove in the cleaning process.

With my outlier-free dataset the model achieved an accuracy of 75.91% and a F1-score of 0.7788.

The fifth model I used was an Adaboost Classifier. This model is also sensitive to outliers, so I used the dataset I already prepared for the LDA model. For the Adaboost Classifier I used GrdSearchCV again to find the optimal parameters. The model achieved an accuracy of 76.66% and a F1-score of 0.7801.

For the last model I tried to combine my best performing models with the SVM, Logistic Regression and Adaboost in a Voting Classifier. Unfortunately this did not work as well and only got an accuracy of 54.89% and a F1-score of 0.6050.

Conclusion

Comparing the accuracy and the F1-scores of my models the best results were achieved by the SVM model using Radial Basic Function and a C parameter of 100 (SVC RBF 2). With this model I can predict if an offer will be successful in nearly 80% of the cases.

Possible Improvement

To further optinmize the model I might train it to be more diverse. For now I can only predict, wether an offer is more likely to be successful. But if I want to personalize my campaigns I want to know which offer would be best suited for what type of customer.
